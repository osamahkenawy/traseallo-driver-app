================================================================================

[P2-1] No 401 → token refresh in axios interceptor
  File : src/api/client.js  (~line 182)
  Risk : Driver kicked to login screen the moment token expires
         mid-shift (on /driver-app/profile we already saw 401 → logout).
  Fix  : Implement refresh-token flow if backend supports; otherwise
         queue-and-retry once per 401 before forcing logout.
  Status: DEFERRED — needs backend refresh-token endpoint. Current
         interceptor already single-fires onAuthExpired callback.

[P2-2] orderId / packageId passed straight into URL paths
  File : src/api/orders.js, src/api/packages.js, src/api/stops.js
  Risk : `undefined`/`null` produces requests to `/orders/undefined`.
  Fix  : Validate (`Number.isFinite(Number(id))`) before request.
  Status: FIXED — added requireId(id, label) helper in client.js;
         applied across orders.js / packages.js / stops.js.
         scanLookup also validates non-empty barcode.

[P2-3] useAuth validateSession has no error path
  File : src/hooks/useAuth.js  (~line 28)
  Risk : Silent rejection on cold start → user stuck on splash.
  Fix  : try/catch; on failure show login.
  Status: FIXED — RootNavigator.bootstrap wraps validateSession in
         try/catch with __DEV__ warn and always sets isReady.

[P2-4] ForgotPasswordScreen interval racing on rapid mount/unmount
  File : src/screens/auth/ForgotPasswordScreen.js  (line 46)
  Fix  : Guard with mounted ref before setState in interval cb.
  Status: VERIFIED OK — uses functional setState + unmount
         clearInterval cleanup; no stale-closure risk.

[P2-5] LocationStore tracking interval may persist if startTracking
        called twice without stopTracking
  File : src/store/locationStore.js  (line 170)
  Note : There IS a `clearInterval(existing)` guard now, but the
         store has no teardown when app is force-quit. Consider
         AppState 'background' handling.
  Status: PARTIAL — useLocation already wires AppState foreground
         ping. Force-quit teardown is OS-level (cannot guard).

[P2-6] No GPS quality validation before sending pings
  File : src/store/locationStore.js  (~line 100)
  Risk : Negative or NaN accuracy values pollute backend tracks.
  Fix  : Reject `accuracy <= 0` or `accuracy > 200`.
  Status: FIXED — sendPing rejects accuracy <= 0 or > 200 m and
         out-of-range lat/lng before POST.

[P2-7] NotificationStore unread-count race
  File : src/store/notificationStore.js  (~line 50)
  Risk : Rapid push events → out-of-order responses → wrong badge.
  Fix  : Debounce 300 ms or track `requestId`.
  Status: FIXED — fetchUnreadCount uses _unreadReqId token; stale
         responses are discarded.

[P2-8] Image upload requests use default 30 s timeout
  File : src/api/uploads.js
  Risk : Large proof-of-delivery photos fail on weak signal.
  Fix  : Pass per-request `timeout: 60_000` and progress handler.
  Status: FIXED — all upload methods set timeout: 60000 and
         expose onUploadProgress callback.
